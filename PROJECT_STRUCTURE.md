## Key Directories Explained

### `/src` - Source Code
**This is where you develop your application.**

- **`index.html`**: Main entry point with all HTML structure, CSS styles, and script includes
- **`js/app.js`**: Application logic, UI controllers, event handlers
- **`js/db.js`**: Database layer using IndexedDB (via `idb` library)
- **`assets/libs/`**: Third-party libraries loaded via script tags

### `/public` - Public Assets
Files here are copied to the root of `dist/` during build.
- `manifest.webmanifest`: PWA configuration for installable web app

### `/dist` - Build Output
**Auto-generated by `npm run build`. Don't edit manually.**
- Optimized and minified production files
- Served by web server or copied to Android project

### `/android` - Native Android Project
**Generated by Capacitor. Use Android Studio to modify.**
- Contains native Android code
- Web assets are synced here from `dist/`
- Build APK from here or via CI/CD

## File Responsibilities

### Core Application Files

#### `src/index.html`
- **Purpose**: Main application structure and UI
- **Contains**: 
  - HTML layout for all sections
  - CSS styles (inline for performance)
  - Script includes for libraries
  - Modal dialogs and popups
- **Sections**:
  - Header (`.hdr`)
  - Login (`.sec#login`)
  - Dashboard (`.sec#dashboard`)
  - Check-in (`.sec#checkin`)
  - Rooms (`.sec#rooms`)
  - Customers (`.sec#customers`)
  - Accounting (`.sec#accounting`)
  - Employees (`.sec#employees`)
  - Settings (`.sec#settings`)
  - Bottom Navigation (`.bnav`)

#### `src/js/app.js`
- **Purpose**: Application logic and controllers
- **Modules**:
  - Network & Connectivity
  - Theme Management (Light/Dark)
  - Authentication & Login
  - Navigation
  - Quick Actions
  - Dashboard
  - Settings
  - Employee Management
  - Utilities
- **Exports**: All functions to `window` object for HTML access

#### `src/js/db.js`
- **Purpose**: Database abstraction layer
- **Technology**: IndexedDB via `idb` library
- **Stores**:
  - `rooms`: Room information and status
  - `customers`: Customer data and history
  - `bookings`: Booking records
  - `transactions`: Financial transactions
  - `settings`: Application settings
  - `employees`: User accounts
  - `images`: Stored images (receipts, slips)
  - `sync`: Sync metadata
- **Features**:
  - CRUD operations for all entities
  - Data export/import
  - Automatic seeding of default data

### Configuration Files

#### `package.json`
```json
{
  "scripts": {

    "build": "vite build",            // Build for production
    "preview": "vite preview",        // Preview production build
    "sync": "npx cap sync",           // Sync to Capacitor

    "android:build": "npx cap build android" // Build APK
  },
  "dependencies": {
    "@capacitor/*": "^7.0.0",         // Mobile app plugins
    "idb": "^8.0.0",                  // IndexedDB wrapper
    "chart.js": "^4.4.0"              // Charts and graphs
  },
  "devDependencies": {
    "vite": "^6.0.0"                  // Build tool
  }
}
```

#### `vite.config.js`
- **Purpose**: Vite build tool configuration
- **Settings**:
  - `root`: Source directory (`src/`)
  - `publicDir`: Public assets directory
  - `build.outDir`: Output directory (`dist/`)
  - `build.target`: ES modules target (`esnext`)
  - `resolve.alias`: Path aliases for imports

#### `capacitor.config.json`
- **Purpose**: Capacitor mobile app configuration
- **Settings**:
  - `appId`: Package identifier
  - `appName`: App display name
  - `webDir`: Web assets directory (`dist/`)
  - `plugins`: Plugin configurations
  - `android`: Android-specific settings

## Development Workflow

### 1. Development Mode
```bash
npm run dev
```
- Starts dev server at `http://localhost:3000`
- Hot module replacement (auto-reload)
- Source maps for debugging
- No minification

### 2. Build for Production
```bash
npm run build
```
- Minifies JavaScript and CSS
- Optimizes assets
- Generates production-ready files in `dist/`
- Adds content hashes for caching

### 3. Sync to Android
```bash
npm run sync
```
- Copies `dist/` to `android/app/src/main/assets/public/`
- Updates Capacitor plugins
- Syncs configuration

### 4. Run on Device
```bash
npm run android
```
- Builds Android app
- Installs on connected device
- Opens in Android Studio if no device connected

### 5. Build APK
```bash
npm run android:build
```
- Creates debug APK
- Output: `android/app/build/outputs/apk/debug/app-debug.apk`

## Best Practices

### File Organization
1. **Keep all source code in `/src`**
2. **Never edit `/dist`** - it's auto-generated
3. **Never edit `/android`** unless adding native plugins
4. **Use ES modules** with `import/export`

### Code Organization
1. **One responsibility per module**
2. **Use JSDoc comments** for documentation
3. **Async/await** for database operations
4. **Error handling** with try/catch

### Naming Conventions
- **Files**: lowercase with hyphens (`my-module.js`)
- **Functions**: camelCase (`loadDashboard()`)
- **Constants**: UPPER_SNAKE_CASE (`DB_NAME`)
- **CSS Classes**: lowercase with hyphens (`.card-title`)
- **IDs**: camelCase (`#loginUser`)

### State Management
- Use `state` object for global state
- Avoid global variables
- Pass parameters to functions
- Return values from functions

## Common Tasks

### Add New Section
1. Add HTML section in `src/index.html`
2. Add navigation button in `.bnav`
3. Add loader function in `src/js/app.js`
4. Update `sectionLoaders` object in `goSec()`

### Add New Database Store
1. Add store name to `STORES` constant in `db.js`
2. Create store in `upgrade()` function
3. Add CRUD operations to `db` object
4. Export new operations

### Add New Capacitor Plugin
1. Install plugin: `npm install @capacitor/plugin-name`
2. Sync: `npx cap sync`
3. Import and use in code

### Update External Libraries
1. Update `package.json` version
2. Run: `npm install`
3. Rebuild: `npm run build`

## Troubleshooting

### Build Fails
```bash
# Clean and rebuild
rm -rf dist node_modules/.vite
npm run build
```

### Sync Fails
```bash
# Force sync
npx cap sync android --force
```

### Database Issues
```javascript
// Clear database in browser console
indexedDB.deleteDatabase('resort-suite-v5')
// Reload page
```

### App Not Loading
1. Check browser console for errors
2. Verify all script paths in HTML
3. Ensure database initializes correctly
4. Check network status

## Performance Tips

1. **Use IndexedDB** for large datasets (not localStorage)
2. **Lazy load** sections (load on demand)
3. **Debounce** search inputs
4. **Virtualize** long lists
5. **Cache** frequently accessed data
6. **Minimize** DOM manipulation
7. **Use CSS** animations (GPU accelerated)

## Security Considerations

1. **Never store** passwords in plain text
2. **Validate** all user inputs
3. **Sanitize** HTML output (prevent XSS)
4. **Use HTTPS** in production
5. **Implement** proper authentication
6. **Limit** data exposure in exports
